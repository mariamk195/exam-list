📖 The Study System — One Pass, Maximum Retention

🎯 Active Recall Method

For each topic, use Explain → Test → Fix:

  • Watch/read the content simultaneously taking notes (you know this works best for you)
  • After each subtopic, close notes and recite key facts out loud or write from memory
  • Compare back to notes, fix gaps immediately — do NOT re-read passively
  • Create one question per subtopic ("How do you differentiate TTP from HUS?") in your cheat sheet

✅ The goal is retrieval, not re-reading. If you can say it without looking, you own it.

⏱️ The Flow: Topic by Topic

  1. Pick 1-2 high-yield topics for the session
  2. Watch/read the source material (Dirty Medicine / Zero to Finals / Sketchy)
  3. Build cheat sheet slide WHILE watching (concurrent — never after)
  4. Cover the cheat sheet → bullet-point recall from memory
  5. Check subtopics off as you truly master each point
  6. Do 3-5 MCQs on that topic immediately (Amboss/Pastest)

🔑 Never move on until you can answer: What is it? How does it present? What do you investigate? How do you treat?

🔁 Spaced Repetition (No Anki Required)

Day 1: Study topics → build cheat sheet slide
Day 3: Cover slide, recall → 5 min review
Day 7: Flip through entire deck, cover answers
Day 14+ (review week): Full deck rapid-fire pass

✅ Your PowerPoint cheat sheet IS your Anki deck. Q on one side, A on the other.

📊 MCQ Strategy

Use past exam patterns as your compass:

  • Pattern recognition: "Fever + neuro + thrombocytopenia + MAHA = TTP → plasma exchange"
  • Do MCQs in tutor mode (see explanation after each Q)
  • For wrong answers: add a red flag bullet to the relevant cheat sheet slide
  • Do at least 5-10 questions per topic after your cheat sheet is made

⚠️ Don't do question banks before making your cheat sheet — you'll guess, not learn.

📄 The Cheat Sheet System — PowerPoint Edition

💡 Why PowerPoint?

  • Fast to build (drag boxes, color code, copy-paste tables)
  • Easy to review on your iPad/phone anywhere
  • Can add images (ECGs, rash photos, CXR) directly
  • Can run a slideshow for rapid self-testing
  • One rotation = One PPT file (5 files total)

✅ 1 SLIDE = 1 TOPIC. Max 1-2 slides per big topic. If it needs more than 2 slides, you're over-noting.

📐 Slide Template (Every Slide)

Top strip: Topic name + yield tag (⭐⭐⭐ HIGH)

Left column (~60%):

  • Pathophysiology in 1 sentence
  • Key clinical features (bullet list, max 5)
  • Investigation of choice (bold + red)
  • Management (numbered steps)

Right column (~40%):

  • Mnemonic or table (e.g., CRAB for MM)
  • Key differentiating feature from DDx
  • 1 "EXAM TRAP" box in red (e.g., "First step = IV fluids NOT bisphosphonates")

🔑 Exam traps are the most important boxes — they're exactly what the MCQs test.

⚡ Speed Build Method

Set a 20-minute timer per slide. If not done — stop anyway and finish during review.

  1. Open blank PPT slide
  2. Paste in the 4-question framework (What/How/Investigate/Treat)
  3. Fill in while source material is still playing
  4. Add ONE color per rotation (red=heme, purple=neuro, orange=EM, blue=ICU, green=peds)
  5. Done? Cover the right half — can you recall the exam traps?

✅ Use a PPT template with the columns pre-built. Set it up once — then duplicate for every topic.

🗂️ Active Recall Using Your Slides

Fold method (analog alternative): Print 2 slides per page, fold in half. Question/title on left, answer on right.

Digital method:

  • Duplicate every slide → Delete answers from duplicate → Put question version first
  • Slideshow mode → Read the question slide → mentally answer → advance to answer slide
  • Works perfectly on iPad in Keynote during commute

⚠️ NEVER just re-read your slides passively. Cover. Recall. Check. That's the only way.
